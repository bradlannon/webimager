<task type="auto">
  <name>Task 1: Create BackgroundControls component and dark mode CSS</name>
  <files>src/components/BackgroundControls.tsx, src/index.css</files>
  <action>
1. **Create `src/components/BackgroundControls.tsx`** that renders different UI states based on the hook's `status`:

   **Import:** `useBackgroundRemoval` hook, `useEditorStore` for `cropMode` and `backgroundRemoved`.

   **State machine rendering (switch on `status`):**

   - **`idle` + not backgroundRemoved:** Show "Remove Background" button with an icon (use `Eraser` or `Wand2` from lucide-react). Disable button when `cropMode` is true (with `disabled:opacity-50 disabled:cursor-not-allowed`). On click: call `requestRemoval()`.

   - **`idle` + backgroundRemoved (i.e., status returned to idle after done):** Show "Restore Background" button. On click: call `restoreBackground()`. Also show a subtle "Remove again" option if user wants to re-run.

   - **`confirming`:** Show inline confirmation message replacing the button:
     - Text: "One-time ~45MB download. Runs entirely in your browser -- your photo never leaves your device."
     - "Download & Continue" button (primary style).
     - "Cancel" text button to go back to idle.
     - On confirm: call `confirmDownload()`.

   - **`downloading`:** Show:
     - Label: "Downloading model..."
     - Progress bar (div with percentage width, use Tailwind: `h-2 bg-blue-500 rounded-full transition-all` inside `h-2 bg-neutral-200 dark:bg-neutral-600 rounded-full`).
     - Progress percentage text.
     - Cancel button. On click: call `cancel()`.

   - **`inferring`:** Show:
     - Label: "Removing background..."
     - Indeterminate/pulsing progress bar (use CSS animation: `animate-pulse` on the bar, or a bar that fills back and forth).
     - Cancel button. On click: call `cancel()`.

   - **`done`:** Show "Restore Background" button (same as idle+backgroundRemoved). The status transitions to show the toggle.

   - **`error`:** Show:
     - Error message text (from `error` state).
     - "Try Again" button. On click: call `requestRemoval()` to restart.

   **Styling:** Follow existing component patterns (Tailwind classes matching TransformControls, AdjustmentControls). Use `text-sm`, neutral color scheme, dark mode variants.

   **When `backgroundRemoved` is true (regardless of status):** Show "Restore Background" button. This is the toggle -- clicking it calls `restoreBackground()` which toggles the flag, showing the original image. The button then changes back to "Remove Background".

2. **Add dark mode checkerboard CSS to `src/index.css`:**
   - Add a `@media (prefers-color-scheme: dark)` block after the existing `.checkerboard-bg` rule.
   - Dark mode colors: `#404040` for the checker squares, `#333333` for the background color. Same gradient pattern, just darker colors.
   - Keep the same `background-size` and `background-position`.
  </action>
