  <files>src/types/editor.ts, src/store/editorStore.ts, src/utils/canvas.ts, src/hooks/useRenderPipeline.ts, src/__tests__/editorStore.test.ts, src/__tests__/renderPipeline.test.ts</files>
  <behavior>
    - editorStore: rotateRight from 0 produces 90, from 90 produces 180, from 180 produces 270, from 270 produces 0
    - editorStore: rotateLeft from 0 produces 270, from 270 produces 180, from 180 produces 90, from 90 produces 0
    - editorStore: flipHorizontal toggles flipH boolean
    - editorStore: flipVertical toggles flipV boolean
    - editorStore: resetAll returns transforms to defaults (rotation: 0, flipH: false, flipV: false)
    - editorStore: setImage stores bitmap reference and resets transforms
    - limitSize: image under 16,777,216 pixels returns original dimensions and wasDownscaled=false
    - limitSize: image over 16,777,216 pixels returns scaled dimensions and wasDownscaled=true
    - limitSize: scaled dimensions maintain aspect ratio
    - renderToCanvas: with rotation=0, flipH=false, flipV=false, canvas dimensions match source
    - renderToCanvas: with rotation=90, canvas width=source.height, canvas height=source.width
  </behavior>
  <action>
1. Create `src/types/editor.ts` with:
   - `Transforms` interface: `{ rotation: 0 | 90 | 180 | 270; flipH: boolean; flipV: boolean }`
   - `EditorState` interface with: sourceImage (ImageBitmap | null), originalFile (File | null), wasDownscaled (boolean), transforms (Transforms)
   - `defaultTransforms` constant exported
   - Export all types

2. Create `src/store/editorStore.ts` following the Zustand store pattern from RESEARCH.md exactly:
   - Import types from `../types/editor`
   - Actions: setImage (closes old bitmap, resets transforms), rotateLeft, rotateRight, flipHorizontal, flipVertical, resetAll
   - Use modular arithmetic for rotation: `((rotation + 90) % 360)` for right, `((rotation - 90 + 360) % 360)` for left

3. Create `src/utils/canvas.ts` with:
   - `MAX_CANVAS_PIXELS = 16_777_216` constant
   - `limitSize(width: number, height: number)`: returns `{ width, height, wasDownscaled }` -- scales down if pixels exceed MAX_CANVAS_PIXELS, preserving aspect ratio
   - `renderToCanvas(ctx, source, transforms)`: applies rotation + flip using canvas transforms, swaps canvas dimensions for 90/270 rotation. Follow the exact pattern from RESEARCH.md.
   - `drawCheckerboard(ctx, width, height, tileSize?)`: draws transparency checkerboard pattern

4. Create `src/hooks/useRenderPipeline.ts`:
   - Takes a canvas ref
   - Subscribes to `useEditorStore` for sourceImage and transforms
   - Calls `renderToCanvas` whenever state changes via useEffect
   - Draws checkerboard behind the image

5. Write tests FIRST (red), then implement (green):
   - `src/__tests__/editorStore.test.ts`: Test all rotation transitions (4 directions x 2 rotate actions), flip toggles, resetAll, setImage resets transforms
   - `src/__tests__/renderPipeline.test.ts`: Test `limitSize()` with under-limit, over-limit, and edge-case dimensions. Test `renderToCanvas()` canvas dimension swapping for rotations. Note: use a mock canvas context (CanvasRenderingContext2D methods mocked) since jsdom has no real canvas. Test that correct transform calls are made (translate, rotate, scale, drawImage).
