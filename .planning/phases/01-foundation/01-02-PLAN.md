  <behavior>
    - loadImage rejects files with types other than image/jpeg, image/png, image/webp
    - loadImage accepts valid image types and returns an ImageBitmap
    - loadImage calls limitSize and downscales if needed, returning wasDownscaled=true
    - loadImage uses createImageBitmap for EXIF auto-correction (default imageOrientation)
    - DropZone prevents default on dragover and drop events (no browser navigation)
  </behavior>
  <action>
1. Create `src/hooks/useImageLoader.ts`:
   - Export async function `loadImage(file: File): Promise<{ bitmap: ImageBitmap; wasDownscaled: boolean }>`
   - Validate file.type against ['image/jpeg', 'image/png', 'image/webp'] -- throw Error with descriptive message if invalid
   - Call `createImageBitmap(file)` -- this auto-corrects EXIF orientation (default `imageOrientation: "from-image"`)
   - Check pixel count against MAX_CANVAS_PIXELS using `limitSize()` from utils/canvas
   - If downscaling needed, call `createImageBitmap(file, { resizeWidth, resizeHeight, resizeQuality: 'high' })`
   - Return `{ bitmap, wasDownscaled }`
   - Export a `useImageLoader()` hook that wraps loadImage with loading/error state and calls `editorStore.setImage` on success

2. Create `src/components/DropZone.tsx`:
   - Full-page drop zone -- this IS the landing page per user decision
   - Large centered text: "Drop your photo here or click to browse"
   - Hidden `<input type="file" accept="image/jpeg,image/png,image/webp">` triggered on click
   - Drag event handlers: onDragEnter, onDragOver, onDragLeave, onDrop -- ALL call `e.preventDefault()` and `e.stopPropagation()`
   - Drag feedback per user decision: dashed border that changes color on drag-over (e.g., `border-blue-500` highlight)
   - On drop: extract file from `e.dataTransfer.files[0]`, call useImageLoader
   - Validation errors appear inline on the drop zone (per user decision) -- no popups or toasts
   - Loading state: show a spinner or "Loading..." text while image processes
   - If wasDownscaled: store flag in editor state (info badge shown by Editor later)
   - Tailwind styling: dark mode aware (`dark:` variants), full viewport height, centered layout
   - Use lucide-react `Upload` icon for visual affordance

3. Write `src/__tests__/imageLoader.test.ts` FIRST (red then green):
   - Test: rejects non-image file (e.g., text/plain) with descriptive error
   - Test: accepts image/jpeg file type
   - Test: accepts image/png file type
   - Test: accepts image/webp file type
   - Note: createImageBitmap is not available in jsdom. Mock it globally in the test file. Mock it to return a fake ImageBitmap-like object with width/height properties and a close() method.
   - Test: when pixels exceed MAX_CANVAS_PIXELS, wasDownscaled is true and createImageBitmap is called with resize options
  </action>
