**BottomBar.tsx** — Icon tab bar with panel management:
- Fixed at bottom, full width, uses `.glass` class, ~60px height (icon + label)
- Border-top with subtle neutral border
- 6 tabs in order: Crop, Transform, Adjustments, Background, Resize, Download
- Each tab: Lucide icon (24px) above text label (text-xs), flex-col centered
  - Icons: Crop, RotateCw (or Repeat), SlidersHorizontal, Eraser, Maximize (or Scaling), Download
  - Choose the clearest Lucide icon for each — these are at Claude's discretion
- Active tab: icon and label colored with accent #2A9D8F (or Tailwind teal-600)
- Inactive tabs: text-neutral-500 dark:text-neutral-400, icon stroke matching
- State management with `useState<string | null>(null)` for `activeTab`
- On tab click:
  - If tapping the already-active tab → set activeTab to null (close panel)
  - If tapping a different tab → set activeTab to that tab's id (switch panel)
- Renders children based on activeTab — the panel content will be passed from Editor in Plan 02
- For now, render a placeholder `<OverlayPanel>` with text like "Panel content here" for each tab, to prove the mechanism works
- Export the component. Also export the `TabId` type (a union type of the 6 tab string ids)

**OverlayPanel.tsx** — Reusable slide-up panel container:
- Props: `open: boolean`, `onClose: () => void`, `children: ReactNode`, optional `className?: string`
- When `open` is true:
  - Render a semi-transparent backdrop overlay (covers the area between top bar and bottom bar) that closes the panel on click
  - Render the panel container sliding up from above the bottom bar
  - Panel uses `.glass-panel` class for frosted glass
  - Rounded top corners (rounded-t-2xl), padding, max-height ~50vh with overflow-y-auto
  - On desktop (min-width: 768px / md breakpoint), panel is max-w-lg centered horizontally (not full width)
  - On mobile, panel spans full width
- Slide animation: Use CSS transitions on transform (translateY). When closed: translateY(100%), when open: translateY(0). Transition duration ~250ms with ease-out
- The backdrop should have a subtle dark overlay (bg-black/20) so the panel stands out
- When `open` is false, do not render anything (or render with pointer-events-none and translateY(100%) for animation-out — prefer the approach that gives smooth close animation)
  - Best approach: always render but use opacity-0 + pointer-events-none + translateY(100%) when closed, opacity-100 + pointer-events-auto + translateY(0) when open. This allows CSS transition on both open and close.
