       - Import `PresetGrid` from `./PresetGrid`
       - Import `useEditorStore` selectors for `sourceImage`, `activePreset`, `setPreset` (in addition to existing imports)
       - Add a `thumbnailUrl` state using `useState<string | null>(null)`
       - Add a `useEffect` that generates a 64px thumbnail data URL from `sourceImage` when it changes:
         ```
         if (!sourceImage) { setThumbnailUrl(null); return; }
         const size = 64;
         const canvas = document.createElement('canvas');
         const aspect = sourceImage.width / sourceImage.height;
         canvas.width = aspect >= 1 ? size : Math.round(size * aspect);
         canvas.height = aspect >= 1 ? Math.round(size / aspect) : size;
         const ctx = canvas.getContext('2d')!;
         ctx.drawImage(sourceImage, 0, 0, canvas.width, canvas.height);
         setThumbnailUrl(canvas.toDataURL('image/jpeg', 0.6));
         ```
       - Render `<PresetGrid thumbnailUrl={thumbnailUrl} activePreset={activePreset} onSelect={setPreset} />` at the TOP of the component's return JSX, before the slider divs
       - Add a thin separator (`<hr className="border-neutral-200" />`) between the preset grid and the sliders

    IMPORTANT: CSS `filter` on HTML `<img>` elements works in all browsers natively (no polyfill needed). The polyfill is only for Canvas 2D `ctx.filter`. So thumbnail previews work everywhere.

    NOTE: Exports already work automatically. renderToCanvas calls buildFilterString(adjustments), which now includes sepia() and hue-rotate() from Plan 01. When a preset is selected, it overrides the adjustments in the store, so the existing export pipeline picks up the preset values with zero changes.
