<task type="auto">
  <name>Task 1: CropOverlay and CropToolbar components</name>
  <files>src/components/CropOverlay.tsx, src/components/CropToolbar.tsx</files>
  <action>
    Create CropOverlay.tsx -- an HTML/CSS layer positioned absolutely over the canvas:

    Props: canvasRect (DOMRect from canvas getBoundingClientRect), sourceWidth/sourceHeight (actual image dims after rotation), aspectRatio (number | null from selected preset), onCropChange (callback with CropRegion).

    Structure:
    - Positioned absolute, same size as the canvas display area
    - Dark semi-transparent overlay with a "hole" for the crop area using CSS clip-path polygon (see RESEARCH.md code example)
    - Crop rectangle border: 2px white/light border for visibility
    - 8 drag handles: small squares (12px desktop, min-w-11/min-h-11 on mobile per user decision of 44px min tap target) at corners and edge midpoints
    - Cursor styles: corner handles get nw-resize/ne-resize/etc, edge handles get n-resize/e-resize/etc, interior gets move cursor

    Drag interaction:
    - Use Pointer Events API (pointerdown, pointermove, pointerup)
    - On pointerdown on a handle: call setPointerCapture(e.pointerId), record start position and start crop
    - On pointermove: calculate delta in display pixels, convert to percentage using displayDeltaToPercent pattern from RESEARCH.md
    - Handle type determines which edges move:
      - 'nw': adjusts x, y, width, height (top-left corner)
      - 'n': adjusts y, height (top edge)
      - 'ne': adjusts width, y, height (top-right corner)
      - 'e': adjusts width (right edge)
      - 'se': adjusts width, height (bottom-right corner)
      - 's': adjusts height (bottom edge)
      - 'sw': adjusts x, width, height (bottom-left corner)
      - 'w': adjusts x, width (left edge)
    - Interior drag: moves x, y without changing width/height
    - After each move: clampCrop to valid bounds, then if aspectRatio is set, apply constrainToAspectRatio
    - On pointerup: release pointer capture, call onCropChange with final crop

    Keyboard: Escape key calls exitCropMode (from store). Enter key calls applyCrop.

    Create CropToolbar.tsx:
    - Positioned above or below the canvas area (in the editor toolbar area)
    - Contains: preset dropdown (select element with CROP_PRESETS options), Apply button (green/primary), Cancel button (secondary)
    - Apply calls applyCrop() from store
    - Cancel calls exitCropMode() then clearCrop() if no previous crop existed, or exitCropMode() to restore previous crop
    - Track whether a crop existed before entering mode to decide cancel behavior
    - Preset selection: when a non-free preset is selected, immediately constrain the current crop region to that ratio and call setCrop. When "Free" is selected, remove ratio constraint.

    Styling: Tailwind classes. Handles use bg-white border border-neutral-400 with hover states. Overlay uses bg-black/50. Crop border is border-2 border-white with shadow for contrast against light images.
  </action>
