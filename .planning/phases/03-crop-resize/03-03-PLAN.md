  <files>src/components/ResizeControls.tsx, src/components/Sidebar.tsx</files>
  <action>
    Create ResizeControls.tsx:

    Local state:
    - targetWidth (number) -- initialized from current image width (post-crop if crop exists)
    - targetHeight (number) -- initialized from current image height (post-crop if crop exists)
    - aspectLocked (boolean, default true)
    - isPercentage (boolean, default false)
    - isApplying (boolean, default false) -- loading state during resize

    Current dimensions: derive from sourceImage dimensions. If cropRegion exists and is not full image, calculate cropped pixel dimensions using cropToPixels. The resize operates on the post-crop result per user decision.

    UI layout:
    - Width input: number type, min=1, max=10000, with "W" or "Width" label
    - Height input: number type, min=1, max=10000, with "H" or "Height" label
    - Lock/Unlock toggle icon between width and height: use Lock/Unlock icons from lucide-react. When locked, changing one input updates the other via calcResizeDimensions. Visual indication: connected line or icon button between inputs.
    - Pixel/Percentage toggle: small button group or toggle. When percentage mode is active, inputs show percentage values (100 = current size). When switching modes, convert current values.
    - Upscale warning: if either targetWidth > currentWidth or targetHeight > currentHeight, show amber/yellow text: "Enlarging may reduce quality"
    - Apply button: calls applyResize(targetWidth, targetHeight) from store. Disabled when dimensions match current or while applying. Shows spinner/loading state during async resize.
    - After resize completes, update targetWidth/targetHeight to reflect new dimensions.

    Width change handler:
    - If aspectLocked, calculate proportional height via calcResizeDimensions
    - If percentage mode, convert percentage to pixels for display in the other field

    Height change handler:
    - If aspectLocked, calculate proportional width via calcResizeDimensions

    Re-initialize inputs when sourceImage changes (via useEffect on sourceImage).

    Styling: Tailwind classes matching existing sidebar patterns. Inputs use the same compact style as existing controls. Lock toggle is a small button between the two inputs.

    Update Sidebar.tsx:
    - Import ResizeControls
    - Add CollapsibleSection "Resize" between Adjustments and Download sections
    - Render ResizeControls inside it
