### Integration Bug

**BGREM-05 Status Desync (user-visible)**
- **File:** `src/hooks/useBackgroundRemoval.ts` lines 144-146
- **Issue:** `restoreBackground` calls `clearBackgroundMask()` but does not call `setStatus('idle')`. After restore, the hook's local `status` remains `'done'`, causing `BackgroundControls` to render "Restore Background" again instead of "Remove Background".
- **Impact:** User cannot re-remove background without page reload.
- **Fix:** Add `setStatus('idle')` in the `restoreBackground` callback.
