Three targeted fixes:

1. **src/hooks/useBackgroundRemoval.ts line 145** — In the `restoreBackground` callback, replace `useEditorStore.getState().toggleBackground()` with `useEditorStore.getState().clearBackgroundMask()`. The `toggleBackground()` action is bidirectional (flips `backgroundRemoved` boolean) which means double-clicking "Restore" silently re-applies removal and leaves a stale mask in memory. `clearBackgroundMask()` already exists in the store and correctly nulls out `backgroundMask`, sets `backgroundRemoved: false`, and clears `replacementColor`.

2. **src/store/editorStore.ts applyResize (line 191-219)** — Two changes needed:
   a. Pass `backgroundMask` and `replacementColor` to the `renderToCanvas` call on line 201. Change from:
      `renderToCanvas(ctx, state.sourceImage, state.transforms, state.adjustments, state.cropRegion ?? undefined)`
      to:
      `renderToCanvas(ctx, state.sourceImage, state.transforms, state.adjustments, state.cropRegion ?? undefined, state.backgroundRemoved ? state.backgroundMask : undefined, state.backgroundRemoved ? state.replacementColor : undefined)`
      This ensures the resized bitmap correctly includes the background removal compositing.
   b. In the `set()` call on lines 211-217, add background state clearing:
      `backgroundRemoved: false, backgroundMask: null, replacementColor: null`
      This prevents stale mask (with wrong dimensions) from persisting after resize.

3. **src/hooks/useRenderPipeline.ts line 3** — Remove `drawCheckerboard` from the import. Change from:
   `import { renderToCanvas, drawCheckerboard } from '../utils/canvas';`
   to:
   `import { renderToCanvas } from '../utils/canvas';`
   The checkerboard is rendered via CSS `.checkerboard-bg` class on the canvas container, making this import dead code.
