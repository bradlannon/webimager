<task type="auto">
  <name>Task 1: Create useBackgroundRemoval hook</name>
  <files>src/hooks/useBackgroundRemoval.ts</files>
  <action>
Create `src/hooks/useBackgroundRemoval.ts` that manages the full worker lifecycle:

**State the hook exposes:**
- `status: 'idle' | 'confirming' | 'downloading' | 'inferring' | 'done' | 'error'`
- `downloadProgress: number` (0-100)
- `error: string | null`
- `modelCached: boolean` (tracks whether model has been loaded at least once in this session)

**Actions the hook exposes:**
- `requestRemoval()`: If `modelCached` is false, sets status to `'confirming'`. If true, goes straight to inference.
- `confirmDownload()`: Creates worker (if not already created), posts `{ type: 'load-model' }`, sets status to `'downloading'`.
- `startInference()`: Extracts source image pixels from store's `sourceImage` using an offscreen canvas (`getSourceImageData` pattern from research), posts `{ type: 'run-inference', imageData }` with transferable buffer. Sets status to `'inferring'`.
- `cancel()`: If status is `'downloading'`, terminates worker and creates fresh one (to abort download). If status is `'inferring'`, sets a `cancelledRef` flag so when result arrives it's discarded. Resets status to `'idle'`.
- `restoreBackground()`: Calls `useEditorStore.getState().toggleBackground()`.

**Worker message handling (in onmessage):**
- `download-progress`: Update `downloadProgress` from `progress` field.
- `model-ready`: Set `modelCached = true`, then immediately call `startInference()`.
- `inference-start`: Set status to `'inferring'`.
- `inference-complete`: If not cancelled, call `useEditorStore.getState().setBackgroundMask(maskData)`. Set status to `'done'`.
- `error`: Set status to `'error'`, store error message.

**Worker lifecycle:**
- Create worker lazily on first `confirmDownload()` using: `new Worker(new URL('../workers/backgroundRemoval.worker.ts', import.meta.url), { type: 'module' })`.
- Store worker in a `useRef<Worker | null>`.
- On component unmount, terminate worker if exists.
- Keep worker alive between operations (model stays loaded in memory).

**Helper function `getSourceImageData`:**
- Takes `sourceImage: ImageBitmap`, creates offscreen canvas at source dimensions, draws image, returns `ctx.getImageData(0, 0, width, height)`.

1. **Modify `src/utils/canvas.ts` -- add `backgroundMask` parameter to `renderToCanvas`:**
   - Add optional `backgroundMask?: ImageData | null` as the last parameter.
   - After the existing image rendering (AFTER ctx.filter is applied and ctx.restore() is called), add mask compositing block:
     ```
     if (backgroundMask) {
       // Create temp canvas with mask, transformed identically to source
       const maskCanvas = document.createElement('canvas');
       maskCanvas.width = ctx.canvas.width;
       maskCanvas.height = ctx.canvas.height;
       const maskCtx = maskCanvas.getContext('2d')!;
       // Put mask as ImageData on temp canvas, then transform it
       // For the no-crop path: apply same rotation/flip/translate as source
       // For the crop path: put mask on offscreen, apply rotation/flip, then extract crop region
       // Then composite: ctx.globalCompositeOperation = 'destination-in'; ctx.drawImage(maskCanvas, 0, 0);
       // Reset: ctx.globalCompositeOperation = 'source-over';
     }
     ```
   - CRITICAL: The mask compositing MUST happen AFTER ctx.filter and ctx.restore() -- this avoids premultiplied alpha fringing (see research Pitfall 1).
   - The mask is at source dimensions. Apply the EXACT same transform pipeline to it (rotation, flip, crop extraction). For the crop path: create an offscreen canvas at rotatedW x rotatedH, put mask ImageData, rotate/flip same as source, then extract the crop region. For the no-crop path: create canvas at rotatedW x rotatedH, put mask, rotate/flip, then draw with destination-in.
   - To put the mask ImageData on a canvas correctly transformed: create a temp canvas at source.width x source.height, putImageData(mask, 0, 0), then draw that temp canvas onto the maskCanvas with the same translate/rotate/scale transforms used for the source image.
