<task type="auto">
  <name>Task 2: Wire zoom/pan into Canvas with wheel events, pointer events, CSS transform, and cursor management</name>
  <files>src/components/Canvas.tsx</files>
  <action>
    Modify `src/components/Canvas.tsx` to add zoom and pan interaction:

    1. **Import zoom utilities and store actions:**
       - Import `zoomAtPoint`, `clampZoom` from `../utils/zoom`
       - Subscribe to `zoomLevel`, `panOffset`, `setZoom`, `setPan`, `resetView` from store

    2. **CSS Transform on wrapper div:**
       - The inner `<div className="relative">` wrapper gets a `style` with CSS `transform: scale(${zoomLevel}) translate(${panOffset.x / zoomLevel}px, ${panOffset.y / zoomLevel}px)` and `transformOrigin: '0 0'`.
       - NOTE: translate values are divided by zoomLevel because translate executes AFTER scale in the combined transform string, but we want pan in screen-space pixels. Alternatively, use `translate(${panOffset.x}px, ${panOffset.y}px) scale(${zoomLevel})` with transformOrigin 0 0 — this applies translate first in screen space, then scale. Use whichever produces correct behavior (translate-then-scale is simpler for pan math).
       - The container div keeps `overflow-hidden` to clip zoomed content.

    3. **Wheel event handler (zoom):**
       - Use `useEffect` to attach native `addEventListener('wheel', handler, { passive: false })` on `containerRef.current` (NOT React onWheel — React 19 makes onWheel passive, preventing preventDefault).
       - Call `e.preventDefault()` to block browser zoom.
       - Calculate mouse position relative to container: `mouseX = e.clientX - rect.left`, `mouseY = e.clientY - rect.top`.
       - Compute zoom factor: if `e.ctrlKey` (trackpad pinch), use `1 - e.deltaY * 0.01`; else (scroll wheel), use `e.deltaY > 0 ? 0.9 : 1.1` for ~10% steps per tick.
       - Compute `newZoom = clampZoom(zoomLevel * zoomFactor)`.
       - Compute `newPan = zoomAtPoint(mouseX, mouseY, zoomLevel, newZoom, panOffset)`.
       - Call `setZoom(newZoom)` and `setPan(newPan)`.
       - Dependencies: `[zoomLevel, panOffset, setZoom, setPan]`.

    4. **Pointer event handlers (pan):**
       - Add a `panDragRef = useRef<{ startX, startY, startPan } | null>(null)`.
       - `onPointerDown` on the container div (NOT the inner wrapper): if `zoomLevel <= 1`, return (no pan at fit-to-view). Set pointer capture, store start position and current panOffset.
       - `onPointerMove`: if panDragRef.current exists, compute delta from start, set pan to `startPan + delta`.
       - `onPointerUp`: release pointer capture, clear panDragRef.
       - PAN BOUNDS: Clamp pan so at least 20% of the scaled image remains visible. Compute bounds based on container dimensions and scaled image dimensions.

    5. **Cursor management:**
       - On the container div, set `cursor` style dynamically:
         - At `zoomLevel <= 1`: `'default'` (no pan available)
         - At `zoomLevel > 1` when NOT dragging: `'grab'`
         - While dragging (panDragRef.current is set): `'grabbing'`
       - During crop mode when zoomed in: cursor defaults to grab (crop handles have their own cursors via CropOverlay).
       - Use a state variable `isPanning` to track active drag for cursor swap.

    6. **Double-click to reset:**
       - Add `onDoubleClick={handleDoubleClick}` on the container div.
       - `handleDoubleClick` calls `resetView()` from store.

    7. **Crop coexistence:**
       - CropOverlay already calls `e.stopPropagation()` on its pointer events. The pan handler is on the container div. When user clicks on crop handles, CropOverlay captures the event and it never reaches the container's pan handler. When user clicks outside crop handles (on the dim overlay or exposed canvas), the event reaches the container and triggers pan.
       - CropOverlay is INSIDE the transformed wrapper div, so it scales with zoom — crop handle positions remain correct.
       - No changes needed to CropOverlay.tsx.

    8. **Touch support for two-finger pinch:**
       - The wheel event handler already handles trackpad pinch (ctrlKey + wheel).
       - For touchscreen: add touch event listeners for two-finger pinch. Use `touchstart/touchmove/touchend` — when `e.touches.length === 2`, compute distance between fingers, compare to initial distance, derive zoom factor. Apply `zoomAtPoint` centered on midpoint of the two fingers.
       - Single-finger drag on touch is already handled by pointer events (pan).
